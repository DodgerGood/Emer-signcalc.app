<analysis>**original_problem_statement:**
Build a multi-tenant web application for wide format signage estimating.
**PRODUCT REQUIREMENTS:**
- **Authentication:** Role-based (Manager, Estimator, Procurement, Quoting Staff, CEO). Initially JWT was chosen.
- **Data Isolation:** Each company (tenant) has isolated data.
- **Core Entities & Management:** Managers can create and manage Materials, Ink Profiles, Labour Types, Install Types, and Travel Settings.
- **Recipes:** Managers can create Recipes which are assemblies of different line item types (MATERIAL, INK, LABOUR, etc.).
- **Quoting:** Estimators can create quotes by selecting a recipe and providing dimensions. The system calculates costs, waste, and markup.
- **Approvals:** Markup overrides and quote-to-job conversion require manager approval.
- **Exports:** PDF for client quotes and Excel (BOM) for internal bill of materials.
- **Recipe Versioning:** New versions should replace old ones, with old versions being archived.
- **UI:** Keep it clean and simple.
- **Currency:** Prices should be in South African Rands (ZAR).
- **Role-Specific Access:**
    - **Manager:** Full access except creating quotes.
    - **Procurement:** Access to materials pricelist only.
    - **Quoting Staff:** Access to quotes and approvals, can convert to job tickets.
    - **CEO:** Full access except creating quotes.
- **Material Types:** Support for different units like SQM, Sheet, Roll, Board, and Unit.
- **Password Management:** Users should have a settings tab to change their password.
- **Session Management:** If a user logs in from a new device, the old session should be terminated.

**User's preferred language**: English
